O que entra na 0.3.1? (marque)

1

Intervalo custom (de → até, data + hora)

Seletor com data e hora de início/fim. Inputs nativos datetime-local — zero-dep, sem libs de calendário.

2

Adicionar preset 30d

Completa a régua (hoje para em 7d, mas uptime já fala em 30d).

3

Range vale para todas as seções do projeto

Escolheu o intervalo numa seção, ele persiste ao navegar entre Requests/Errors/Queries/… (via query string + cookie).

4

Logs: filtro combinado (texto + nível + intervalo)

A seção Logs ganha barra única: buscar texto E nível E a janela temporal específica juntos.

🔍 "payment failed" · error · 14/06 09:00→18:00
5

Busca de log no banco (não só nos recentes)

Hoje a busca de log filtra só o lote recente em memória. Para achar log de uma data passada, a query precisa ir ao banco (respeitando retenção/prune). Mais robusto, requer cuidado de índice.

Recomendação: 1+2+3+4 são o coração do pedido e baratos. O item 5 é o que realmente entrega "achar log de uma data antiga" — vale incluir, mas é o de maior custo (query no banco + índice em occurred_at); se o escopo apertar, ele é o candidato a 0.3.2.
